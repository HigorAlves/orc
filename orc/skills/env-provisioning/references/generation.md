# Generation — templates and rules for orc-owned compose files

Everything generated lands in `.orc/<branch>/files/docker/` (gitignored via `.orc/`). The repo tree stays clean — `git status` must be empty after provisioning. Adoption into the repo (a real committed compose file) happens only on explicit user request through the normal plan → PR path.

## Hybrid mode (rung-4 default): services-only compose

App runs on the host via its dev script; only backing services are containerized. Fastest boot, zero invented app packaging, native file-watching.

```yaml
# .orc/<branch>/files/docker/compose.orc.yaml — generated by orc; do not commit
name: <project>            # orc-<name>-<sanitized-branch>
services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: app
      POSTGRES_PASSWORD: app
      POSTGRES_DB: app_dev
    ports: ["5432:5432"]
    volumes: [db_data:/var/lib/postgresql/data]
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U app"]
      interval: 2s
      timeout: 3s
      retries: 30
volumes:
  db_data:                 # named per-project → survives down, reused next boot
```

Host side: run the dev script with `env.orc` applied (`set -a; . env.orc; set +a; pnpm dev`), record the PID + logfile in `hostProcesses[]` alongside `services` in state. Wire env values to the containers (`DATABASE_URL=postgres://app:app@localhost:5432/app_dev`).

## Full-container mode (`--containerize-app`, or rungs 2–3)

Adds an app service. Without a project Dockerfile (rung 4 forced): generic node image + bind-mounted worktree + **named `node_modules` volume** (bind-mount node_modules is the classic macOS perf killer):

```yaml
  app:
    image: node:22-alpine        # version from .nvmrc / engines.node
    working_dir: /app
    command: sh -c "corepack enable && pnpm install && pnpm dev"
    volumes:
      - /abs/path/to/worktree:/app        # the WORKTREE, not the main checkout
      - app_node_modules:/app/node_modules
    env_file: [env.orc]
    ports: ["3000:3000"]
    depends_on:
      db: { condition: service_healthy }
    healthcheck:
      test: ["CMD-SHELL", "wget -qO- http://localhost:3000 || exit 1"]
      interval: 5s
      timeout: 5s
      retries: 60
      start_period: 30s
```

With a project Dockerfile (rung 3): `build: { context: <worktree>, dockerfile: Dockerfile, target: <dev stage if multi-stage> }` instead of `image:`. BuildKit is the compose-v2 default — never disable it; see `docker-expert/references/multistage-builds.md` for cache-mount patterns.

## Workspace wrapper

```yaml
# .orc/<branch>/files/docker/compose.workspace.yaml
name: orc-<ws>-<branch>
include:
  - path: /abs/<ws>/api/docker-compose.yml     # repos with their own compose
    project_directory: /abs/<ws>/.orc/.worktrees/api/<branch>
services:
  ui:                                           # repos without compose: generated block
    # …generated app service as above, depends_on the api service
```

`include:` needs compose ≥ 2.20 (`docker compose version`). Fallback when unavailable or service names collide: per-repo projects on one shared external network (`docker network create orc-<ws>-<branch>-net`; each compose gets an override attaching to it); state records `composeFiles[]` per repo + the shared `network`.

Dependency order: `depends_on: {condition: service_healthy}` edges from the plan's cross-repo contract when present, else env-var references (ui's env mentions `API_URL` → ui depends on api). Record the resolved order in state `dependencyOrder`.

## Port-conflict remaps

Only generated files may be remapped. Pick the next free port (`lsof -i :<port>`), write an override:

```yaml
# compose.orc-override.yaml
services:
  db: { ports: !override ["55432:5432"] }
```

Update `serviceEndpoints` + `env.orc` to match. For a rung-1 project compose with a foreign port conflict, never remap silently — gate (stop other env / remap via override / abort) since the project's own docs assume its ports.

## Naming and hygiene rules

- Project name: `orc_env_project_name` (lowercase, `[a-z0-9_-]`, ≤ 63 chars). Volumes inherit the project prefix automatically.
- Every generated service MUST carry a healthcheck (`up --wait` depends on it).
- `env.orc` never contains invented secrets — placeholders + a surfaced warning.
- Generated YAML must pass `docker compose config` before any boot; the rendered output is the `compose.rendered.yaml` evidence artifact.
